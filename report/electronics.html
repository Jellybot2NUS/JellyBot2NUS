1. Overview of the Electronics System

The electronics subsystem serves as the control and sensory core of Jellybot, enabling communication, data processing, and coordination across all hardware modules. It integrates the microcontroller, sensor interfaces, and signal conditioning circuits that collect and process environmental data for navigation and control. Sensor communication is managed through digital protocols such as I²C and UART, with dedicated routines for data filtering and fusion to ensure stable readings. The subsystem is designed with waterproof connectors, organized PCB layouts, and proper grounding to minimize electrical noise and ensure reliability under high-pressure and high-humidity conditions.

The listed components form the core sensing and control framework of Jellybot. Each sensor contributes a unique data stream: the MPU6050 IMU provides orientation through accelerometer and gyroscope readings, the HMC5883L magnetometer offers heading information, the SEN0311 ultrasonic sensor measures distance to obstacles, and the camera module delivers real-time visual feedback for monitoring and analysis. These modules communicate through I²C, UART, and CSI interfaces, all managed by the microcontroller. Together, they enable Jellybot to perceive its environment, support navigation and stabilization algorithms, and provide essential feedback for control. The total power and mass remain minimal, maintaining system efficiency and suitability for underwater deployment.

3. Wiring Diagram

The wiring diagram illustrates the interconnection between the sensors, controller, and peripheral devices that make up the Jellybot’s electronics subsystem. An external PS5 controller functions as the primary user interface, allowing manual control and navigation input during testing. The Raspberry Pi Camera Module 2 is directly mounted on the Raspberry Pi 5 through the CSI interface, providing live visual feedback for monitoring. The SEN0311 ultrasonic sensor communicates with the Raspberry Pi via UART, enabling accurate distance measurement, while both the MPU6050 IMU and HMC5883L magnetometer share the I²C bus for synchronized orientation and heading data acquisition. This configuration ensures efficient communication, minimal wiring complexity, and reliable real-time data exchange across all connected modules.

Findings & Limitations :

The captured images were clearest and most distinguishable at distances between 0.5 m and 1.0 m , aligning with theoretical expectations for underwater imaging resolution. Beyond this range, image clarity declined noticeably due to light scattering and refraction in the water. However, as the test utilized a simplified prop and controlled tank environment, the results may not fully replicate the complexity of actual coral reef textures or the variability of real underwater conditions such as turbidity and lighting. These preliminary findings nonetheless provide a practical reference for determining the optimal camera-to-subject distance in future field tests.

Future Testing Plans :

Future tests will focus on improving environmental realism and quantitative image analysis. Printed images will be used to better replicate natural reef textures. Still images will be captured at fixed intervals to evaluate sharpness and distortion across varying distances. Additionally, MATLAB Edge-Detection techniques will be employed to analyze image quality, focus, and detection reliability under different underwater conditions. These refinements aim to enhance the accuracy of camera calibration and the relevance of test results to real-world applications.

4.2 Navigation Testing

Motivation for Magnetometer :

Using only a 6-axis IMU (accelerometer + gyroscope) does not provide a reliable long-term heading. Yaw from a gyro is obtained by integrating angular rate, so even a small bias accumulates over time and causes drift. To obtain an absolute reference for heading, a 3-axis magnetometer is added. The magnetometer measures Earth’s magnetic field, giving a global reference for yaw; combined with accelerometer-derived roll/pitch (tilt compensation), it yields a stable compass-like heading.

The accelerometer and magnetometer data are fused mathematically to derive the orientation angles. The following equations summarize the key computations used in this system:

  • Accelerometer Conversion: Raw sensor readings are scaled using the ±2g range factor.
    ax = AXRAW / 16384, ay = AYRAW / 16384, az = AZRAW / 16384
  • Roll and Pitch from Accelerometer:
    roll = atan2(ay, az)
    pitch = atan2(-ax, √(ay² + az²))
  • Magnetometer Calibration (Hard-Iron Offset):
    m'x = mx - bx, m'y = my - by, m'z = mz - bz
  • Tilt Compensation: The magnetic vector is rotated into the horizontal plane to remove the influence of pitch and roll.
    mhx = m'xcos(pitch) + m'zsin(pitch)
    mhy = m'xsin(roll)sin(pitch) + m'ycos(roll) - m'zsin(roll)cos(pitch)
  • Heading Calculation:
    heading = atan2(mhy, mhx)
    Converted to degrees and normalized to 0–360°.
  • True Heading (Optional): Local magnetic declination can be added for geographic correction.
    true_heading = (heading + declination) % 360
  • Magnetic Field Magnitude: Used to detect nearby magnetic interference.
    |B| = √(m'x² + m'y² + m'z²)

This approach ensures that Jellybot maintains a consistent directional reference even in the presence of gyroscopic drift. The fusion of accelerometer and magnetometer data forms the foundation of the compass-like heading used for navigation and orientation stability underwater.

Objective :

The objective of this test was to evaluate the JellyBot’s ability to determine its orientation accurately, achieving a compass-like heading using the onboard magnetometer. Reliable heading information is crucial for navigation and control in complex underwater environments.

Test Setup :

The test employed the JellyBot’s compass and magnetometer sensors. Care was taken to keep the test area clear of magnets and other electronic devices that could introduce interference and affect readings. The JellyBot was rotated through multiple headings while stationary to assess the sensor’s responsiveness and accuracy.

Findings :

The magnetometer generally provided readings that aligned with the expected headings, confirming that the sensor functioned according to its specifications. The system successfully demonstrated basic orientation detection capability.

Limitations :

Despite the overall functionality, the magnetometer readings tended to drift over time and were occasionally inconsistent. Minor residual magnetic interference in the environment contributed to these fluctuations, highlighting the need for better isolation of the sensor.

Future Testing Plans :

  • Setup: Improve the testing environment to eliminate all sources of magnetic interference, including hidden electronic components and nearby ferrous materials.
  • Measurements: Record heading data over extended periods and across multiple orientations to assess long-term stability.
  • Analysis: Implement calibration and filtering algorithms to reduce drift and enhance measurement consistency.

Conclusions :

The preliminary navigation tests verified the magnetometer’s basic functionality and established a baseline for orientation detection. Further refinements in sensor placement, environmental isolation, and data processing will improve heading accuracy and reliability for precise underwater navigation.

5. User Journey (Controller)

This section illustrates the interface of the JellyBot controller, designed to provide the user with a seamless and intuitive operational experience. The controller interface is intended to guide the user through key stages of deployment and monitoring, ensuring that the robot is ready for underwater exploration before activation.

Upon powering on the controller, the Welcome Menu appears, providing an overview of the system and access to the User Guide, which offers instructions for operating the JellyBot, including safety precautions and setup procedures.

Once the preliminary checks are complete, the Ready to Deploy screen confirms the operational status of the JellyBot. The interface continuously monitors the connection between the controller and the robot, as well as the status of onboard sensors, to ensure that the system is fully functional before deployment.

Ready to deploy

During operation, the Exploration Menu allows users to navigate the JellyBot, monitor sensor data, and control movements for reef exploration. If the controller is not properly connected to the robot, a Controller Not Connected pop-up alerts the user to check the connection. Similarly, if any onboard sensor is not detected or functioning, a Sensor Not Connected notification provides real-time feedback, preventing deployment until the issue is resolved.

6. Summary and Future Improvements

The electronics subsystem effectively manages signal processing, data acquisition, and communication between sensors and control modules. It forms the core intelligence that enables Jellybot’s autonomous functions and real-time responsiveness. Future iterations could integrate more compact embedded processors, improved sensor fusion techniques, and enhanced waterproofing strategies to increase performance, reduce latency, and support more advanced underwater analytics.
