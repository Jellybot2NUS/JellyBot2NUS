01

Overview

The electronics subsystem of Jellybot is built around the Blue Robotics Navigator Flight Controller HAT, mounted on a Raspberry Pi 4 Model B. Unlike a conventional ROV where the flight controller directly drives motors, the Navigator in this project serves exclusively as a sensor and attitude data source. Motor and solenoid control is handled by a separate custom system, which preserves the Navigator's role purely for telemetry, orientation sensing, and depth measurement.

The software stack runs on BlueOS [a Docker-based middleware operating system] with ArduSub 4.5.3 STABLE providing firmware-level sensor fusion. Communication between subsystems uses the MAVLink protocol, translated to HTTP/JSON by MAVLink2REST, enabling browser-based tools and custom extensions for live sensor data.

The major contribution of the electronics subsystem this semester is driven by the characterisation of optical distortion introduced by the dome and customising BlueOS Docker extensions.

02

Hardware

2.1 Sensors and Actuators

The listed components form the core sensing and control framework of Jellybot. Each sensor contributes a unique data stream:

    • To provide a user control interface [design specification 3b], the Playstation 4 Controller was selected.
    • To measure distance to obstacles [design specification 4a], the SEN0311 Ultrasonic Sensor was selected.
    • To provide orientation [design specification 4b], the MPU6050 IMU was selected.
    • To provide heading information [design specification 4c], the HMC5883L Magnetometer was selected.
    • To provide real-time visual feedback for monitoring and analysis [design specification 4d], the Raspberry Pi Camera Module V2 was selected.
    • To coordinate communication between all onboard sensors [design specification 5], the Raspberry Pi 5 was selected.
